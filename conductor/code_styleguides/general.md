# 一般的なコードスタイル原則

このドキュメントでは、このプロジェクトで使用されるすべての言語とフレームワークに適用される一般的なコーディング原則の概要を説明します。

## 読みやすさ
- コードは人間が読みやすく、理解しやすいものであるべきです。
- 過度に巧妙または難解な構成は避けてください。

## 一貫性
- コードベース内の既存のパターンに従ってください。
- フォーマット、命名、構造の一貫性を維持してください。

## 単純さ
- 複雑な解決策よりも単純な解決策を優先してください。
- 複雑な問題を、管理しやすい小さな部分に分割してください。

## 保守性
- 変更や拡張が容易なコードを記述してください。
- 依存関係と結合を最小限に抑えてください。

## ドキュメンテーション
- *何が*行われたかだけでなく、*なぜ*行われたのかを文書化してください。
- ドキュメントをコードの変更に合わせて最新の状態に保ってください。
